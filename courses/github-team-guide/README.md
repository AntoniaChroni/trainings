# GitHub Best Practices — Team Guide

Please take a moment to review the sections below. Together, they outline our shared expectations for clear communication, effective code review, and productive collaboration on GitHub. All files are located at `./resources`.

---

## Code Review Communication Guidelines (The 7 Cs)

This section explains **how to communicate during code reviews**. It focuses on clarity, conciseness, correctness, and respect, helping reviewers provide feedback that is easy to understand, actionable, and constructive—regardless of seniority.

---

## Pull Request (PR) Review Checklist — Code Reviewers

This checklist provides **practical guidance for reviewing pull requests**, from understanding the intent of a PR to leaving actionable, well‑scoped comments. It is designed for **both junior and senior reviewers**, supporting consistency, learning, and high‑quality reviews across the team.

---

## How to Report Bugs on GitHub

This section outlines **best practices for reporting bugs** so issues are reproducible, well‑documented, and easy to triage. Clear bug reports help maintainers diagnose problems faster and ensure issues are addressed efficiently.
