# 📌 How to Report Bugs on GitHub

Please take a moment to double‑check your configuration and parameters before submitting a bug report. Ensuring that required parameters are correctly updated will prevent unnecessary confusion and help us focus on true bugs.
 
Accurate bug reports are essential for efficient debugging and for keeping the pipeline stable for all users. When reporting an issue, please make sure to:

- Review the documentation for the relevant module
- Confirm that all YAML parameters are correctly set
- Add an informative title and include the exact parameters, error messages, and steps to reproduce, e.g. https://github.com/stjude-dnb-binfcore/sc-rna-seq-snap/issues/226
- Share logs or screenshots when applicable

This level of detail helps us identify legitimate bugs much more quickly and prevents misinterpretation of expected behavior as code defects.
 
Thank you for your collaboration and for helping keep the workflows reliable for everyone.  If anything is unclear or you need help interpreting a parameter, always reach out to the developer.
